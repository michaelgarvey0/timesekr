📝 Design Notes
  • Mobile-First: All screens should be designed for mobile first, then adapt to tablet/desktop
  • Email Templates: Screens 11-14 are email views but need corresponding web landing pages
  • State Management: Consider empty states, loading states, and error states for each screen
  • Onboarding Skip Logic: If user skips calendar/contacts, re-prompt at appropriate moments
  • Responsive Design: Time grid (Screen 10) needs special attention for mobile view
  • Accessibility: Ensure all interactive elements meet WCAG 2.1 AA standards
✅ Screen Design Priority Order
  1. Screen 1-5: Core authentication and navigation (Week 1)
  2. Screen 6-10: Meeting creation flow (Week 2)
  3. Screen 11-12: Email invitation templates (Week 3)
  4. Screen 13-16: Advanced features (Week 4)
  5. Screen 17-19: Enhancement screens (Week 5+)
⚙️ Technical Considerations
  • OAuth Integration: Screens 1, 3, 4 require third-party auth flows
  • Calendar APIs: Screen 3 connects to Apple, Google, Microsoft, Yahoo calendars
  • Email Rendering: Screens 11-14 must work across all major email clients
  • Deep Linking: Email CTAs should deep link to app if installed, web otherwise
  • Time Zone Handling: Critical for Screen 8, 10 - display local time for each user
